Notes
  • MessageService (shared-services): The single source of truth for all messages (info, warning, error, success)
  • MessageListenerService (shared-ui): Listens to MessageService and displays notifications in the UI
  • NotificationDisplayService (shared-ui): Internal service that handles the actual UI rendering
  • All messages go through MessageService - it's the only service that should be used to show messages
  • Details Dialog: Error and warning messages with details show "(click for details)" and open a modal dialog when clicked
  • Copy to Clipboard: The details dialog includes a button to copy detailed information to clipboard
  • Methods: showSuccess(message), showInfo(message), showErrorWithDetails(title, details) and showWarningWithDetails(title, details)
  • Messages are automatically logged to console based on their level
  • Error messages are stored for history/debugging and can be retrieved
  • Auto-hide behavior: Success and Info messages auto-close after 3 seconds. Error and Warning messages persist until user interaction or navigation.
  • User interaction: Error/Warning notifications close when clicking anywhere (except navigation) or navigating to another page
  • Clean architecture: MessageService handles business logic, UI services handle presentation only
