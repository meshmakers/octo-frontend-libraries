Runtime Browser Demo

Description

The Field Filter Editor component allows you to create and manage multiple field filters for querying data. Each filter consists of an attribute path, an operator, and a comparison value. The input field type adapts based on the attribute's data type (text, number, boolean, datetime).

Usage

Themes

The component does support custom themes. In order to use custom look & feel adjusted to CI of the file, you can overwrite following CSS variables at the root of your application: 

          {{ themeCssVariables }}
Internationalization

The component does support internationalization. In order to use internationalization using @ngx-translate library. The library must be provided by the host app. Also, the library must be initialized in Angular configuration: 

          {{ i18nProvidersExample }}

After that, the component can be used with the language input (or en-GB by default): 

          {{ runtimeBrowserUsageExample }}

If above conditions were not met, component does fallback to default language: en-GB.

In order to get any translations (including default), you need to update **angular.json** file to include the translations for the host app: 

          {{ angularJsonTranslationsAssets }}
